Data Labeling Workflow

Excel 数据 + 打标规则 → AI 自动生成代码执行打标。基于 DeepSeek V4。

目录结构

E:/Condex/数据处理工作流/
├── input/
│ ├── 主数据表.xlsx // 主表 待打标的核心数据
│ ├── 评论表.xlsx  // 辅表 补充数据（可选）
│ ├── 打标规则.xlsx // 规则 告诉AI怎么打标
│ ├── task_config.txt // 配置 指定文件名
│ └── api.txt    // API 密钥配置
├── runs/
│ └── YYYYMMDD_HHMMSS/
│   ├── output/  // 打标后的文件
│   ├── generated/ // AI生成的代码
│   └── run.log  // 运行日志
├── data_labeling.py // 主程序
└── run_labeling.bat // 双击启动

快速开始

  1. 准备数据：把 Excel 文件放到 input/ 目录。多个文件时，第一个文件是主表（输出以此命名），其余是辅表（自动按共同字段 LEFT JOIN 合并到主表）。
  2. 写规则：用 Excel 或 .md 编写打标规则，放到 input/ 。规则越详细越好（字段名、取值、逻辑、优先级、边界情况）。
  3. 配 task_config.txt：告诉程序文件叫什么名字（见下方示例）。
  4. 配 API：在 input/api.txt 中填入你的 DeepSeek 密钥。
  5. 运行：双击 run_labeling.bat。

配置文件说明

task_config.txt

data_file: "主数据表.xlsx"      "<-- 第一行是主表，最重要"
data_file: "评论数据.xlsx"     "<-- 第二行以后是辅表，自动合并到主表"
requirement_file: "打标规则.xlsx"  "<-- 规则文件，仅一个"

第一个 data_file 是主表，输出文件以它命名。辅表会按共同字段（如 帖子链接）自动 LEFT JOIN 到主表上。

api.txt

api_base = https://api.deepseek.com/v1
model = deepseek-chat
api_key = 删除这段，填入你的deepseekv4 api

支持任何 OpenAI 兼容接口，换服务商只需改 api_base 和 model。

工作流原理

阶段说明
Phase 1读取主表的表头+前5行、辅表的表头+前5行、规则文件的全部内容
Phase 2把规则压缩成结构化格式（保留全部细节）
Phase 3数据预览 + 规则 → DeepSeek，生成打标 Python 代码
Phase 4执行代码，出错自动修复（本地+AI，最多3次），输出打标文件
输出：打标后的文件包含原始所有列 + 新增的中文标签列。每次运行独立存档，不会覆盖。

常见问题

输出列名是中文还是英文？

中文，按规则文件里的字段名生成。

支持 CSV 吗？

支持 .csv 和 .xlsx。

怎么切换项目？

替换 input/ 里的文件 + 改 task_config.txt，不用改代码。

打标结果不好？

优化规则文件。AI 吃的是 prompt，规则写得越清楚结果越好。

可以用 GPT 或 Claude 吗？

可以，改 api.txt 里的 api_base 和 model。兼容 OpenAI 格式的都能用。
